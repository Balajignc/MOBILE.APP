# DevOps Summary

## One-paragraph summary

DevOps is a practice that connects development and operations through automation and shared responsibility. Git is used for version control, Jenkins automates CI/CD pipelines, Terraform provisions infrastructure, Ansible configures systems, Docker creates portable containers, Kubernetes orchestrates those containers, and monitoring plus logging provide visibility into application and infrastructure health.

## Topic summary

- Version Control: tracks code changes and supports collaboration
- Jenkins: automates build, test, and deployment workflows
- Ansible: configures and manages servers
- Terraform: creates infrastructure using code
- Docker: packages apps into consistent containers
- Kubernetes: deploys and scales containers
- Monitoring and Logging: detect issues and support troubleshooting

## Simple real-world flow

1. A developer pushes code to Git
2. Jenkins triggers a pipeline
3. Tests and builds run automatically
4. Docker creates the application image
5. Terraform prepares infrastructure
6. Ansible configures the environment
7. Kubernetes deploys the containerized app
8. Monitoring and logging track system behavior
